# AI_RULES.md

## AI 使用原则

本项目后续可以接入大模型，但 AI 的定位不是替代运营者直接拍脑袋生成内容，而是辅助做分析、判断、推荐和沉淀。

AI 应该服务于这个流程：

```text
账号定位
用户痛点
热点机会
历史作品表现
选题推荐
脚本生成
发布复盘
素材库更新
```

## 当前阶段

当前项目已经接入 Tavily 网页搜索、抖音视频搜索和 DeepSeek 大模型能力，但不是所有页面内容都来自实时 AI。

首页能力需要按来源区分：

- **实时 AI 研判**：点击“生成今日选题”等按钮后，调用抖音搜索、Tavily 搜索和 DeepSeek 生成结果；点击首页“今日刷新”时，仅基于抖音搜索和 DeepSeek 更新首页推荐。
- **规则统计**：播放、收藏、转化、可系列化等复盘卡片，主要来自 `content.json` 中已有数据和前端规则计算。
- **静态默认展示**：首页默认推荐、指标卡、工作流、资产库说明来自项目内 JSON，作为没有刷新前的默认工作台内容。

对外或对用户描述时，不要把所有首页内容笼统称为实时 AI 分析；只有实际触发联网搜索和大模型链路的结果，才标注为 AI 研判或 AI 生成。

首页“今日刷新”返回的联网来源应展示给运营者核对；首页推荐选题只有在用户点击确认后，才允许写入正式选题池。

## 适合接入 AI 的能力

### AI 复盘助手

输入：

- 作品标题
- 脚本或文案
- 平台
- 播放
- 点赞
- 收藏
- 评论
- 转化
- 评论摘要

输出：

- 表现判断
- 表现好的原因
- 表现差的原因
- 可复用结构
- 下次优化方向
- 下一条推荐选题

### AI 热点匹配助手

输入：

- 热点标题
- 热点背景
- 时间窗口
- 账号定位
- 目标用户
- 产品资料

输出：

- 是否适合当前账号
- 适合的目标用户
- 内容角度
- 推荐平台
- 脚本方向
- 风险提醒

### AI 选题扩展助手

输入：

- 一个核心选题
- 目标用户
- 内容栏目
- 业务关联

输出：

- 5 到 10 个系列选题
- 每条选题的核心观点
- 适合平台
- 内容形式
- 产品关联

### AI 脚本生成助手

输入：

- 选题
- 目标用户
- 用户痛点
- 内容形式
- 脚本模板

输出：

- 开头钩子
- 正文结构
- 产品关联
- 评论区引导
- 可拍摄画面建议

### AI 素材沉淀助手

输入：

- 历史作品
- 复盘结论
- 评论摘要
- 产品资料

输出：

- 可沉淀的金句
- 用户痛点
- 可复用案例
- 可扩展选题
- 需要补充的产品资料

## 不建议 AI 做的事

- 不要直接生成脱离账号定位的泛流量标题。
- 不要把内容写成普通消费者种草，而忽略 B 端经营价值。
- 不要在没有产品资料支撑时编造产品优势。
- 不要自动写回素材库，除非用户明确确认。
- 不要把热点强行套到账号上。
- 不要把 AI 输出当成最终结论，运营者需要人工筛选。

## 推荐输出格式

AI 输出应尽量结构化，便于写入系统。

选题推荐建议格式：

```json
{
  "title": "选题标题",
  "targetUser": "目标用户",
  "painPoint": "用户痛点",
  "angle": "内容角度",
  "coreView": "核心观点",
  "businessLink": "业务关联",
  "platform": "推荐平台",
  "format": "内容形式"
}
```

复盘建议格式：

```json
{
  "performance": "表现判断",
  "reason": "原因分析",
  "reusablePattern": "可复用结构",
  "nextAction": "下一步动作",
  "materialUpdates": ["建议沉淀的素材"]
}
```

热点匹配建议格式：

```json
{
  "isSuitable": true,
  "matchedUser": "适合用户",
  "recommendedAngle": "推荐角度",
  "risk": "风险提醒",
  "topicIdeas": ["可延展选题"]
}
```

## 人工确认规则

凡是 AI 生成的内容，如果要进入正式素材库，必须经过人工确认。

需要人工确认的内容包括：

- 选题是否符合 B 端定位
- 产品关联是否真实
- 经营建议是否可落地
- 数据复盘是否合理
- 是否存在夸大或误导表达

## 后续接入建议

第一步：只做 AI 复盘助手。

第二步：增加 AI 热点匹配助手。

第三步：增加 AI 选题扩展和脚本生成。

第四步：增加一键沉淀素材，但必须保留人工确认。

如果要真正保存 AI 结果，需要先选择保存方案：

- localStorage
- 文件写回
- 数据库
- 飞书/Notion/Airtable
