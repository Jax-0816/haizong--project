# AGENTS.md

本文件是海哥内容工作台的 AI 智能体约束文档。所有 AI 助手（Claude、Cursor、Codex 等）在开始任何任务前，必须遵守以下规则。

---

## 1. 任务启动：必须先读文档

AI 开始任何任务前，必须优先阅读：

1. `README.md` — 项目说明、启动方式、结构概览
2. `AGENTS.md` — 本文档，AI 行为约束
3. `TODO.md` — 当前进度和待办事项

如果三个文件内容有冲突，优先级如下：

**用户当前明确指令 > AGENTS.md > README.md > TODO.md**

当任务涉及数据修改、AI 生成内容或新增字段时，还需阅读：

- `docs/AI_RULES.md` — AI 内容生成原则和输出格式约束
- `docs/DATA_SCHEMA.md` — 数据结构和校验规则

---

## 2. 基本行为规则

1. **先理解需求，再查看文件，再给修改方案。** 不要跳过理解直接动手。
2. **修改前必须先说明计划。** 不要直接改代码。
3. **不要修改无关文件。** 只改和当前任务直接相关的文件。
4. **不要新增不必要依赖。** 保持依赖简洁。
5. **不要删除已有功能。** 除非用户明确要求。
6. **不要顺手重构。** 即使看到可改进的地方，先询问而不是直接改。
7. **不确定时先问，不要猜。** 需求不清楚时主动确认。
8. **每次只处理当前任务。** 完成一件再做下一件。

---

## 3. 禁止修改的文件和目录

除非用户明确要求，否则**绝对不能**修改以下内容：

```
node_modules/
dist/
build/
package-lock.json
pnpm-lock.yaml
yarn.lock
.env
.env.local
.env.*.local
```

**安全红线：**

- 不要泄露或写入任何真实密钥、Token、账号、密码、数据库连接信息。
- 不要把真实 API Key 写进 `.env.example`、README 或任何代码文件。
- `.gitignore` 已排除 `.env.local`、`node_modules` 和 `dist`，不要破坏这个规则。

---

## 4. 修改前必须说明

修改任何文件前，必须先向用户说明：

| 事项 | 说明 |
| --- | --- |
| **任务理解** | 你理解用户要做什么 |
| **准备修改的文件** | 列出所有将改动的文件 |
| **修改原因** | 为什么要改这些文件 |
| **是否影响已有功能** | 评估影响范围 |
| **测试方式** | 如何验证修改正确 |

用户确认后，再开始修改。

---

修改完成后，必须按以下格式输出：

```
## 已完成

2. [完成项 2]

## 修改文件

## 测试方式

## 风险提示
```

## 8. 项目特有约束

### 8.1 内容原则

详见 `docs/AI_RULES.md`，核心要点：

- AI 用来做调研、判断、推荐和沉淀建议，**不替代人工最终决策**。
- 不要把内容写成普通消费者种草，要保持 **B 端经营价值**。
- 不要在缺少产品资料时**编造产品优势**。
- AI 结果进入正式素材库、选题池或复盘结论前，**必须人工确认**。

### 8.2 数据约束

详见 `docs/DATA_SCHEMA.md`，核心要点：

- 优先维护 `src/data/content.json`。
- 新增字段时同步更新 `src/types.ts`、使用该字段的组件、以及相关文档。
- 数组项的 `id` 必须稳定，不能用标题当 ID。

### 8.3 开发约束

- 每次修改后至少运行 `npm run build`。
- 修改首页、导航或调研页面后，刷新 `http://127.0.0.1:4280/` 做人工检查。
- 如果 4280 端口被旧服务占用，先停止旧 node 进程，再运行 `npm run dev`。
- 当前 `src/App.tsx` 已经较大，后续新增复杂功能时优先拆组件。

### 8.4 文件优先级（读取顺序）

```
README.md → AGENTS.md → TODO.md → docs/AI_RULES.md → docs/DATA_SCHEMA.md
```
